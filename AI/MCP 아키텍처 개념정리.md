### 2. 구체적인 동작 원리 (The Protocol Handshake)
두 지점이 규격을 맞춘다는 것은 실제 코드 레벨에서 다음과 같은 약속을 지킨다는 뜻입니다.
**1** **초기화 (Handshake):** 호스트가 서버에 "너는 어떤 기능을 가지고 있어?"라고 물으면, 서버는 규격화된 JSON 형식으로 자신의 기능 목록(List Tools)을 보냅니다.
**2** **추론 (Inference):** 호스트는 이 목록을 **LLM**에게 보여줍니다. 모델은 "아, 이 작업을 하려면 tavily_search 도구가 필요하네!"라고 판단합니다.
**3** **실행 (Execution):** 호스트가 모델의 결정을 받아 서버에 규격화된 요청(Call Tool)을 보냅니다.
**4** **응답 (Response):** 서버는 작업 수행 후 다시 규격에 맞는 JSON 응답을 보냅니다.

**3\. MCP 장점(Decoupling)**
**서버 개발자:** "나는 MCP 규격만 맞춰서 서버를 짜두면, 내 서버를 가져다 쓰는 사람이 클로드를 쓰든, GPT를 쓰든 상관없어."
**LLM 서비스 제공자:** "우리는 MCP 인터페이스만 뚫어두면, 전 세계 개발자들이 미리 만들어둔 수만 개의 MCP 서버 기능을 우리 모델에서 바로 쓸 수 있어."

### 5. 둘 사이의 상호작용 흐름 (Flow)
서브 에이전트나 도구 호출이 일어날 때, 두 장치는 다음과 같이 협업합니다.
**1** **Host → LLM:** "사용자가 '오늘 삼성전자 주가 알려줘'라고 했어. 네가 쓸 수 있는 도구 목록은 [tavily_search, mysql_query]야. 어떻게 할래?"
**2** **LLM → Host:** (판단 후) "응, 그럼 tavily_search 도구에 query='삼성전자 주가' 넣어서 실행해줘." (이것이 **Tool Use** 요청입니다.)
**3** **Host → MCP Server:** (실제 실행) MCP 서버에 접속해서 검색을 수행하고 결과를 받아옵니다.
**4** **Host → LLM:** "서버에서 온 결과야: [삼성전자: 75,000원]. 이걸 보고 사용자한테 대답해줘."
**5** **LLM → Host:** (최종 답변) "네, 삼성전자 현재 주가는 75,000원입니다."
