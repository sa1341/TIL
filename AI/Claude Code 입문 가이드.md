# Claude Code 입문 가이드

Claude Code는 터미널에서 파일을 직접 열고 수정하고 명령까지 실행하는 AI 코딩 도구입니다. 잘 쓰려면 **컨텍스트를 아끼고, 계획을 먼저 세우고, 반복 작업을 명령어로 묶는** 세 가지 습관이 핵심입니다.

![Claude Code 작업 흐름 — 컨텍스트 · Plan → Edit|697](../Attached%20file/ai_claude_code_workflow.svg)

## 1. 프로젝트 초기화 및 컨텍스트 최적화

- **루트 디렉터리 실행 필수:** Claude Code는 실행된 위치를 기준으로 파일을 탐색합니다. 특정 모듈 내부가 아닌 프로젝트의 최상위(Root)에서 실행해야 전체 구조를 정확히 파악하고 토큰을 효율적으로 사용합니다.
- **`CLAUDE.md`의 전략적 활용:** `/init` 명령어로 생성되는 이 파일은 프로젝트의 '헌법'과 같습니다.
    - **내용 구성:** 절대 규칙(예: 운영 DB 쿼리 금지), 아키텍처 트리, 빌드/테스트 명령어, 코딩 컨벤션 등을 명시하세요.
    - **경량화 유지:** 루트의 `CLAUDE.md`는 **300라인 이내**로 유지하는 것이 좋습니다. 내용이 많아지면 폴더별(예: `apps/api/CLAUDE.md`)로 분산하는 '레이어드 컨텍스트' 전략을 추천합니다.
    - **자동 업데이트:** 특정 패턴을 정했다면 "방금 정한 패턴을 `CLAUDE.md`에 추가해줘"라고 명령해 자동으로 문서화할 수 있습니다.

## 2. 생산성을 높이는 핵심 단축키 및 모드

- **Plan 모드 vs Edit 모드 (`Shift + Tab`)**
    - **Plan 모드:** 파일 수정 권한 없이 계획만 세웁니다. 불필요한 코드 생성을 막아 **토큰과 비용을 획기적으로 절약**할 수 있습니다.
    - **Edit 모드:** 계획 검토 후 실제 코드를 작성할 때 사용합니다. 항상 **Plan 선행 후 Edit**으로 넘어가는 워크플로우가 권장됩니다.
- **작업 중단 및 리와인드 (`Esc`):** 잘못된 방향으로 코딩할 때 `Esc`로 즉시 중단하고, `Esc`를 두 번 누르면 이전 프롬프트 상태로 되감기(Rewind)가 가능합니다.
- **터미널 내 배시 실행 (`!`):** Claude Code 세션을 나가지 않고도 `!ls`, `!git status`, `!./gradlew build` 등을 바로 실행할 수 있어 컨텍스트 스위칭을 최소화합니다.

## 3. 필수 슬래시(/) 명령어 가이드

- **컨텍스트 관리:**
    - `/context`: 현재 사용 중인 토큰량과 점유율을 확인합니다.
    - `/clear`: 대화 히스토리를 완전히 삭제해 토큰 공간을 확보합니다.
    - `/compact`: 대화 내용은 요약해 남기되, 상세 컨텍스트를 비워 효율을 높입니다.
- **세션 복구:**
    - `/resume`(또는 CLI에서 `claude --resume` · `-r`): 이전에 작업하던 세션을 리스트에서 선택해 그대로 이어갈 수 있습니다. 방금 전 세션을 바로 잇고 싶다면 `claude --continue`(`-c`)를 씁니다.
- **모델 선택 (`/model`):**
    - `Opus`: 가장 똑똑하며 복잡한 아키텍처 설계에 적합합니다.
    - `Sonnet`: 일반적인 코딩이나 반복 작업에 효율적입니다.
    - `Haiku`: 가장 빠르고 저렴해 단순 확인용으로 씁니다.

## 4. 고급 사용자용 커스텀 기능

- **트리거 키워드 (Trigger Keywords):** `CLAUDE.md`에 특정 단어(예: '배포해줘')와 작업 순서를 등록해두면, 해당 단어 입력 시 테스트부터 푸시까지 일괄 실행합니다.
- **커스텀 슬래시 명령어:** `~/.claude/commands/` 폴더에 마크다운 파일을 만들어 나만의 명령어를 정의할 수 있습니다(예: `/ship` 실행 시 테스트 → 커밋 → 푸시 자동화). 트리거 키워드보다 구조적으로 안정적이며 자동완성을 지원합니다.

## 정리

> Claude Code의 핵심은 **컨텍스트 관리**입니다. 루트에서 실행해 전체 구조를 파악시키고, `CLAUDE.md`로 규칙을 고정하며, **Plan으로 먼저 설계하고 Edit으로 구현**하는 흐름을 지키면 토큰과 비용을 아끼면서 품질을 유지할 수 있습니다.
