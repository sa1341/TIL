# CPU와 메모리

컴퓨터의 핵심은 **명령어를 실행하는 CPU**와 **명령어·데이터가 올라오는 메모리**, 그리고 둘을 잇는 **버스**입니다. 흔히 요리사(CPU)와 작업대(메모리)에 비유합니다.

![CPU의 구성과 메모리·버스](../../../Attached%20file/os_cpu_structure.svg)

## 하드웨어 사양 관련 용어

- **클록(Clock)**: CPU의 속도와 관련된 단위입니다. CPU는 작업을 할 때 일정한 박자가 있는데, 이 박자를 만들어내는 것이 클록입니다. 클록이 일정 간격으로 틱(tick)을 만들면 거기에 맞추어 CPU 안의 모든 구성 부품이 작업을 합니다. 틱은 펄스 또는 클록틱이라고도 부릅니다. 버스에도 여러 부품이 연결되어 있어, 메인보드의 클록이 틱을 보낼 때마다 데이터를 보내거나 받습니다.

- **헤르츠(Hz)**: 클록틱이 발생하는 속도를 나타내는 단위입니다. 1초에 클록틱이 몇 번 발생하는지를 나타내며, 1초에 한 번이면 1Hz, 1000번이면 1kHz입니다. 예를 들어 3.4GHz CPU는 1초에 클록틱이 약 34억 번 발생합니다. 버스도 클록틱에 맞추어 데이터를 전송하기 때문에 메인보드의 성능도 클록으로 표시합니다.

> 참고: 클록이 빠르다고 반드시 연산이 그만큼 많아지는 것은 아닙니다. 현대 CPU는 한 클록에 여러 명령어를 처리(파이프라이닝·슈퍼스칼라)하기도 하고, 코어 수도 성능에 큰 영향을 줍니다.

## CPU의 구성과 동작

CPU는 명령어를 해석하여 실행하는 장치로, 요리사 모형에서는 요리사에 해당합니다. CPU는 **산술논리 연산장치, 제어장치, 레지스터**로 구성되며, 이것들의 협업으로 작업을 처리합니다.

- **산술논리 연산장치(ALU)**: 요리사가 재료를 삶거나 찌거나 볶듯이, CPU에서 데이터를 연산하는 장치입니다. 덧셈·뺄셈·곱셈·나눗셈 같은 산술 연산과 AND·OR 같은 논리 연산을 수행합니다.

- **제어장치(Control Unit)**: "도마 위의 재료를 가져와라", "다 된 요리를 접시에 담아라"처럼 각 부품에 작업을 지시하는 부분입니다. 명령어를 해석해 무엇을 할지 결정합니다.

- **레지스터(Register)**: 자른 쇠고기, 껍질 깐 양파처럼 그날 쓸 재료를 미리 손질해 두었다가 필요한 만큼 쥐어 냄비에 넣듯이, CPU 내부에 데이터를 임시로 보관하는 **초고속 저장소**입니다. 대표적으로 다음 실행할 명령어의 주소를 가리키는 **프로그램 카운터(PC)**, 현재 처리 중인 명령어를 담는 **명령어 레지스터(IR)** 등이 있습니다.

## 명령어 사이클 (Fetch → Decode → Execute)

CPU는 아래 세 단계를 쉼 없이 반복하며 프로그램을 실행합니다.

1. **가져오기(Fetch)**: 메모리에서 다음 명령어를 CPU로 가져옵니다. (프로그램 카운터가 그 위치를 가리킴)
2. **해석(Decode)**: 제어장치가 명령어를 해석해 무슨 동작인지 파악합니다.
3. **실행(Execute)**: ALU가 실제 연산을 수행하고, 결과를 레지스터나 메모리에 저장합니다.

## 왜 메모리가 필요한가 — 속도의 계층

CPU(레지스터)는 아주 빠르지만 용량이 작고, 저장장치(HDD/SSD)는 용량이 크지만 느립니다. 그래서 그 중간에서 **실행할 프로그램과 데이터를 잠깐 올려두는 공간이 메모리(RAM)**입니다. CPU는 저장장치를 직접 다루는 대신, 메모리에 올라온 명령어를 하나씩 가져와 처리합니다.

> 속도 순서: **레지스터 > 캐시 > 메모리(RAM) > SSD/HDD**. 위로 갈수록 빠르지만 비싸고 작습니다.
