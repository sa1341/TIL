# 오라클 리스너 (Oracle Listener)

오라클 리스너는 네트워크를 통해 **클라이언트가 오라클 서버에 접속하도록 연결을 받아주는 프로세스**입니다. 네트워크를 통한 모든 접속 요청은 리스너가 받아, 실제 오라클 서버(인스턴스)와 연결해 줍니다. 식당의 "안내 데스크"에 비유할 수 있습니다.

## 접속 과정

1. 클라이언트에 오라클 클라이언트(Oracle Net, 과거 명칭 NET8)가 설치되어 있어야 합니다.
2. 클라이언트는 **서비스명(Service Name)**을 통해 접속할 대상을 지정합니다. (보통 `tnsnames.ora`에 정의)
3. 요청이 서버의 리스너에 도착하면, 리스너가 해당 서비스의 오라클 인스턴스로 연결을 넘겨줍니다.

```
클라이언트  ──(서비스명)──▶  리스너(기본 1521 포트)  ──▶  오라클 인스턴스
```

## 리스너 관리

- 오라클 서버에서 **리스너가 실행 중이어야** 클라이언트가 접속할 수 있습니다.
- `lsnrctl` 명령어로 리스너를 관리합니다.

```sh
lsnrctl start      # 리스너 시작
lsnrctl stop       # 리스너 중지
lsnrctl status     # 상태 확인 (등록된 서비스, 포트 등)
```

> 접속 오류(예: `ORA-12541: TNS:no listener`)는 대부분 리스너가 죽어 있거나, 포트/서비스명이 맞지 않을 때 발생합니다.
