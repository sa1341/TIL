# 배치가 두 번 실행됐다 (크론잡 중복 실행의 함정)

월말 정산 배치가 끝났는데 **금액이 정확히 두 배**가 나왔습니다. 코드에 버그는 없었습니다. **배치가 두 번 실행**된 것이죠. 이전 실행이 안 끝났는데 다음 스케줄이 시작된 것입니다.

## 크론은 이전 실행을 확인하지 않는다

크론잡은 정해진 시간에 작업을 실행합니다. 그런데 **이전 실행이 아직 끝나지 않았을 때 다음 실행 시간이 오면** 어떻게 될까요?

- **리눅스 크론**은 이전 실행이 끝났는지 확인하지 않습니다. 시간이 되면 **무조건 새 프로세스**를 띄웁니다.
- **쿠버네티스 CronJob**도 마찬가지입니다. `concurrencyPolicy` 기본값이 **`Allow`** — 이전 잡이 돌고 있어도 새 잡을 허용한다는 뜻입니다.

두 잡이 같은 테이블에 INSERT하면 데이터가 중복되고, 같은 파일에 쓰면 내용이 꼬입니다.

## 막는 법 — 그런데 이것도 함정

- **리눅스**에서는 **`flock`**으로 막습니다.
- **쿠버네티스**에서는 `concurrencyPolicy`를 **`Forbid`**로 바꾸면 됩니다.

그런데 `Forbid`도 완벽하지 않습니다. **이전 잡이 hang(행)에 걸리면, 다음 잡이 영원히 스킵**됩니다. 배치가 중복 실행되는 것도 문제지만, **아예 안 도는 것도 문제**입니다.

## 실무 — Forbid + activeDeadlineSeconds

그래서 실무에서는 **`Forbid` + `activeDeadlineSeconds`**를 같이 겁니다.

- 일정 시간이 지나면 **hang 걸린 잡을 강제 종료**해서, 다음 스케줄이 돌 수 있게 합니다.

## 정리

> 크론잡의 기본값은 **중복을 허용**하고, 중복을 막으면(`Forbid`) **hang에 취약**해집니다. 스케줄러를 믿지 말고, **방어 설정(`flock` / `Forbid` + `activeDeadlineSeconds`)을 직접** 걸어야 합니다.
